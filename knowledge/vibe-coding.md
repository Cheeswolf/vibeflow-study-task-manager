# 黑马程序员 Vibe Coding 网课学习总结

## 一、学习背景

在学习这门课程之前，我虽然接触过一些计算机和编程知识，也使用 Claude Code 做过项目，但整体技术基础仍然比较薄弱。

如果脱离 AI 工具，让我独立完成一个真实的软件开发项目，我在需求拆解、代码实现、调试、测试、版本管理和项目部署等方面仍然存在明显不足。

Vibe Coding 第一次真正降低了软件开发的技术门槛。它让我意识到，即使暂时不具备扎实的编程能力，也可以通过与 AI 编程工具持续沟通，将一个想法逐步转化为可以运行的产品。

这并不意味着编程知识不再重要，而是让我可以先进入真实开发场景，在项目实践中理解技术、发现问题并积累经验。

## 二、对 Vibe Coding 的初步理解

课程介绍了一种非常纯粹的 Vibe Coding 开发方式，可以概括为一个“黄金循环”：

1. 开发者向 Claude Code 描述需求；
    
2. Claude Code 根据需求生成或修改产品；
    
3. 开发者运行并检查结果；
    
4. 如果产品符合预期，结束本轮开发；
    
5. 如果产品不符合预期，继续与 Claude Code 沟通、修改和测试；
    
6. 重复以上过程，直到产品达到满意状态。
    

这个循环看起来简单，但它实际上要求开发者能够不断观察产品、发现问题、描述问题，并判断最终结果是否符合需求。

因此，Vibe Coding 并不只是“让 AI 帮我写代码”，而是开发者通过自然语言，对软件开发过程进行持续控制。

## 三、开发环境与基础工具

课程介绍了 Vibe Coding 开发需要使用的一些基础环境和工具，包括：

- Node.js；
    
- Python；
    
- Git；
    
- Claude Code。
    

这一部分对我来说难度相对较低，因为我在学习课程之前已经配置并使用过 Claude Code，也曾经通过 Claude Code 完成过一些项目。

不过，这部分内容仍然帮助我重新梳理了开发环境之间的关系，也让我意识到：环境配置、依赖管理和工具使用本身就是软件开发能力的一部分。

## 四、Vibe Coding 入门案例

课程通过多个小型案例帮助学习者熟悉 Claude Code 的基本开发方式，包括：

### 1. 贪吃蛇 GUI 游戏

通过开发贪吃蛇游戏，学习如何向 Claude Code 描述游戏规则、界面需求和交互逻辑，并通过运行结果持续修改程序。

### 2. 黑马记账桌面 App

通过开发桌面记账应用，进一步理解一个相对完整的软件产品通常需要包含的功能，例如数据录入、数据展示、用户操作和界面设计。

这些案例的重点并不只是完成具体产品，而是帮助学习者熟悉从需求描述、代码生成、运行测试到持续修改的基本循环。

## 五、工程化控制

课程后半部分逐渐从“让 AI 生成代码”，进入“如何控制一个持续开发的工程项目”。

### 1. Git 初始化与版本存档

在项目开始时初始化 Git，并在关键开发节点提交代码。

Git 提交相当于为项目建立一个个可恢复的存档点。当后续开发出现严重问题时，可以回到之前相对稳定的版本。

### 2. 上下文压缩

随着对话和项目规模不断扩大，Claude Code 的上下文会逐渐变长。

通过上下文压缩，可以保留重要信息，减少无关内容对后续开发的干扰，也能够降低上下文消耗。

### 3. Memory 记忆管理

通过 Memory 保存项目中长期有效的信息，例如：

- 项目目标；
    
- 技术栈；
    
- 代码规范；
    
- 目录结构；
    
- 开发约束；
    
- 测试要求。
    

Memory 可以减少开发者反复向 Claude Code 解释项目背景的问题。

### 4. 新功能开发与再次存档

在添加新功能之前先确认当前版本稳定，完成新功能后再次进行测试和 Git 提交。

这样可以将每一次功能开发划分成相对独立、可以追踪的阶段。

### 5. Rewind 回退

当 Claude Code 的修改方向错误，或者代码出现较严重问题时，可以利用 Rewind 回到之前的状态，避免继续在错误的基础上修改。

### 6. 推送远程代码仓库

课程使用 Gitee 保存代码，我在实际项目中可以使用 GitHub。

将代码推送到远程仓库，不仅能够进行备份，也方便项目展示、协作和版本追踪。

### 7. 创建分支与代码审查

通过 Git 分支，可以将新功能开发与稳定版本隔离。

功能开发完成后，再进行代码审查和合并，能够减少未经验证的代码直接影响主项目。

### 8. 自定义 Skills

通过 Skills 将常用的开发方法、规则和任务流程封装起来，让 Claude Code 可以重复执行相对标准化的工作。

Skills 使 AI 编程从临时对话逐渐转向可复用、可管理的开发能力。

## 六、Agent 与自动化协作

课程进一步介绍了 Agent 在软件开发中的应用。

### 1. Agent 的基本概念

Agent 不只是回答问题，而是能够围绕一个目标，自主调用工具、执行任务、检查结果并继续行动。

### 2. 单元测试 Skill

通过开发单元测试 Skill，让 Claude Code 可以按照固定流程生成测试、运行测试并分析错误。

### 3. 测试专家 Subagent

创建专门负责测试工作的数字员工，使其能够独立检查功能、发现问题并输出测试结果。

### 4. 质量工程师 Subagent

创建质量工程师角色，对项目的代码质量、稳定性和工程规范进行检查。

### 5. Hook 工具调用拦截

通过自定义 Hook，对 Claude Code 的工具调用过程进行检查和控制。

Hook 可以在某些操作执行前进行拦截，例如阻止危险命令、限制不合理的文件修改，或者要求执行必要的检查。

这一部分让我看到，Vibe Coding 不只是一个人与 AI 的对话，也可以逐渐形成多个 AI 角色分工合作的开发流程。

## 七、RAG 真实项目实践

课程最后通过一个相对真实的 RAG 项目，将前面的知识组合起来。

项目开发过程主要包括：

### 1. 方案讨论与初版代码生成

先与 Claude Code 讨论需求、技术方案和项目结构，再生成第一版可以运行的代码。

### 2. 单元测试与 Bug 调试

对项目核心功能编写单元测试，通过测试结果定位问题，并持续修改代码。

### 3. 迁移 Skills 和 Agent

将之前开发的测试 Skill、测试专家 Agent 和质量工程师 Agent 迁移到新项目中，验证这些能力是否可以跨项目复用。

### 4. Git 提交与版本管理

在关键开发阶段进行 Git 提交，保存稳定版本，记录项目演进过程。

### 5. 并发压力测试

通过压力测试观察系统在多个请求同时访问时的性能和稳定性。

这一部分让我认识到，一个真实的软件项目不仅需要“功能能够运行”，还需要考虑测试、性能、稳定性、版本管理和工程规范。

## 八、个人学习收获

这门课程让我第一次比较完整地看到了一名具备工程思维的开发者，如何使用 Vibe Coding 完成软件开发。

我过去使用 Claude Code 时，更多是在遇到问题后直接让 AI 修改代码。虽然也能够完成项目，但整体开发过程比较随意，缺乏稳定的版本管理、测试流程和上下文管理。

这门课程让我认识到，专业的 Vibe Coding 仍然需要：

- 清晰地描述需求；
    
- 合理地拆分任务；
    
- 管理项目上下文；
    
- 使用 Git 保存版本；
    
- 通过测试验证结果；
    
- 审查 AI 生成的代码；
    
- 建立可以复用的 Skills 和 Agent；
    
- 对产品最终质量负责。
    

因此，Vibe Coding 并不是完全放弃编程和工程知识，而是通过 AI 降低开发门槛，让学习者能够更早进入真实项目，在实践中逐步建立技术理解和工程能力。

目前我已经对课程的整体框架形成了初步认识，但由于只完整观看了一遍，很多命令、配置方法和工程细节还没有真正掌握。

接下来的学习重点不应该只是重复观看课程，而是通过项目复现、主动练习和独立应用，将课程知识转化为自己的开发能力。
